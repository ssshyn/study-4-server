## 1. 왜 리눅스인가

```
서버 OS 시장 점유율:
  Linux: ~96%
  Windows Server: ~3%
  기타: ~1%

이유:
  무료 (라이선스 비용 없음)
  안정성 (수개월 ~ 수년 무중단 운영)
  오픈소스 (문제 발생 시 커널까지 분석 가능)
  Docker/쿠버네티스 등 대부분의 인프라 도구가 Linux 기반

백엔드 개발자가 리눅스를 알아야 하는 이유:
  - 운영 서버에 SSH로 접속해서 문제 분석
  - Docker 컨테이너 내부 진입
  - 배포 스크립트 작성
  - 로그 파일 검색 & 분석
  - 프로세스 및 리소스 상태 확인
```

## 2. 파일시스템 구조

```
/ (루트)
├── bin/     → 기본 명령어 바이너리 (ls, cat, grep...)
├── sbin/    → 시스템 관리 명령어 (root 전용, ifconfig, iptables...)
├── etc/     → 설정 파일 모음
│   ├── nginx/       → Nginx 설정
│   ├── mysql/       → MySQL 설정
│   ├── hosts        → 로컬 DNS (/etc/hosts)
│   └── crontab      → 스케줄 작업
├── var/     → 변하는 데이터 (로그, 캐시, PID 파일)
│   ├── log/         → 시스템 로그
│   │   ├── syslog   → 시스템 이벤트
│   │   ├── auth.log → 인증 로그 (SSH 로그인 시도 등)
│   │   └── nginx/   → Nginx 액세스/에러 로그
│   └── lib/         → 서비스 데이터 (MySQL 데이터 등)
├── tmp/     → 임시 파일 (재시작 시 삭제)
├── home/    → 일반 사용자 홈 디렉토리 (/home/ubuntu, /home/ec2-user)
├── root/    → root 사용자 홈
├── proc/    → 프로세스 정보 (가상 파일시스템, 실제 파일 아님)
│   ├── cpuinfo  → CPU 정보
│   └── meminfo  → 메모리 정보
├── usr/     → 사용자 설치 프로그램
│   └── local/bin/ → 수동 설치한 바이너리
└── opt/     → 서드파티 애플리케이션 설치 위치
```

## 3. 필수 명령어

### 3-1. 파일/디렉토리 탐색

```bash
# 현재 위치
pwd
# /home/ubuntu

# 디렉토리 이동
cd /var/log          # 절대 경로
cd ../               # 상위 디렉토리
cd ~                 # 홈 디렉토리
cd -                 # 바로 이전 디렉토리

# 파일 목록
ls                   # 기본
ls -l                # 상세 (권한, 크기, 날짜)
ls -la               # 숨김 파일(.으로 시작) 포함
ls -lh               # 사람이 읽기 쉬운 크기 (KB, MB)
ls -lt               # 수정 시각 기준 정렬

# 파일 읽기
cat /etc/hosts                    # 전체 출력
head -20 /var/log/syslog          # 앞 20줄
tail -50 /var/log/app.log         # 뒤 50줄
tail -f /var/log/app.log          # 실시간 스트리밍 (Ctrl+C로 종료)
tail -f /var/log/app.log | grep ERROR  # 실시간 에러만 필터링
less /var/log/syslog              # 페이지 단위 조회 (q로 종료)

# 파일 검색
find /var/log -name "*.log" -mtime -1    # 1일 이내 수정된 .log 파일
find /app -name "*.jar" -size +100M      # 100MB 이상 jar 파일
find /tmp -type f -mtime +7 -delete     # 7일 이상 지난 파일 삭제
```

### 3-2. 텍스트 처리 — 로그 분석의 핵심

```bash
# grep — 패턴 검색
grep "ERROR" app.log                     # ERROR 포함 줄
grep -i "exception" app.log             # 대소문자 무시
grep -n "OutOfMemory" app.log           # 줄 번호 포함
grep -v "DEBUG" app.log                 # DEBUG 제외
grep -A 5 "FATAL" app.log               # FATAL 이후 5줄 함께 출력
grep -B 3 "NullPointerException" app.log # 3줄 이전도 출력
grep -c "ERROR" app.log                 # 매칭된 줄 수
grep -r "config" /etc/nginx/            # 디렉토리 재귀 검색

# 여러 조건 조합
grep "ERROR" app.log | grep -v "404"    # ERROR이면서 404 아닌 것
grep "PaymentService" app.log | grep "ERROR"  # PaymentService 에러만

# awk — 컬럼 추출 & 집계
awk '{print $1}' access.log             # 첫 번째 컬럼 (IP 주소)
awk '{print $9}' access.log             # HTTP 상태 코드 컬럼
awk '$9 >= 500' access.log              # 500 이상 상태 코드 줄만
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head  # IP별 요청 수 Top 10

# sed — 텍스트 치환
sed 's/ERROR/[ERROR]/g' app.log         # ERROR를 [ERROR]로 치환
sed -n '100,200p' app.log               # 100~200줄만 출력
sed '/DEBUG/d' app.log                  # DEBUG 포함 줄 삭제

# sort / uniq — 정렬 & 중복 제거
sort app.log                            # 알파벳 정렬
sort -n numbers.txt                     # 숫자 정렬
sort -rn numbers.txt                    # 역순 숫자 정렬
sort -k2 app.log                        # 2번째 컬럼 기준 정렬
uniq access.log                         # 연속된 중복 제거
sort access.log | uniq -c              # 중복 횟수 포함

# wc — 카운트
wc -l app.log                          # 줄 수
wc -c app.log                          # 바이트 수
grep "ERROR" app.log | wc -l          # ERROR 줄 수 세기

# 파이프 조합 실전 예시
# 오늘 날짜 ERROR 로그를 심각도별로 세기
grep "2024-01-15" app.log | grep "ERROR" | awk '{print $5}' | sort | uniq -c | sort -rn

# 가장 많이 요청된 API Top 10
awk '{print $7}' access.log | sort | uniq -c | sort -rn | head -10

# 응답 시간 분석 (awk로 평균 계산)
awk '{sum += $NF; count++} END {print "평균:", sum/count, "ms"}' timing.log
```

### 3-3. 프로세스 관리

```bash
# 실행 중인 프로세스
ps aux                              # 전체 프로세스 목록
ps aux | grep java                  # Java 프로세스만
ps aux --sort=-%cpu | head          # CPU 사용률 높은 순
ps aux --sort=-%mem | head          # 메모리 사용률 높은 순

# 실시간 모니터링
top                                 # 기본 (q로 종료)
# top 안에서:
#   k: PID 입력해서 프로세스 종료
#   1: CPU 코어별 사용률 보기
#   M: 메모리 사용률 기준 정렬
#   P: CPU 기준 정렬

htop                                # 더 편한 top (설치 필요: apt install htop)

# 프로세스 종료
kill -15 {PID}                      # SIGTERM (정상 종료 요청)
kill -9 {PID}                       # SIGKILL (강제 종료, 마지막 수단)
pkill -f "myapp"                    # 이름으로 종료
kill $(cat /var/run/myapp.pid)      # PID 파일로 종료

# 백그라운드 실행
nohup java -jar app.jar > app.log 2>&1 &
# nohup: 터미널 종료해도 계속 실행
# 2>&1: stderr를 stdout으로 합침
# &: 백그라운드 실행

jobs            # 백그라운드 작업 목록
fg 1            # 1번 작업을 포그라운드로
bg 1            # 1번 작업을 백그라운드로
```

### 3-4. 시스템 리소스 확인

```bash
# CPU
nproc                               # CPU 코어 수
cat /proc/cpuinfo | grep "model name" | head -1  # CPU 모델
uptime                              # 부하 평균 (1분, 5분, 15분)
# load average: 0.5, 0.7, 0.6
# 코어 수와 비교: 코어 4개면 4.0이 100% 부하

# 메모리
free -h                             # 메모리 사용량
# total: 16GB, used: 8GB, free: 2GB, buff/cache: 6GB
# 실제 여유 = free + buff/cache (캐시는 필요 시 즉시 반납)
cat /proc/meminfo                   # 상세 메모리 정보

# 디스크
df -h                               # 파일시스템별 용량
df -h /                             # 루트 파티션
du -sh /var/log/*                   # 각 로그 파일 크기
du -sh /home/* | sort -rh          # 홈 디렉토리별 크기
lsblk                               # 블록 디바이스 (디스크) 목록

# 디스크 I/O
iostat -x 1 3                       # I/O 통계 (1초 간격 3회)
iotop                               # 프로세스별 I/O (root 권한)
```

### 3-5. 네트워크 확인

```bash
# 네트워크 인터페이스
ip addr show                        # IP 주소 목록 (ifconfig 대신)
ip addr show eth0                   # 특정 인터페이스
hostname -I                         # 서버 IP 주소

# 포트 확인
ss -tlnp                            # 열려 있는 TCP 포트 목록
# netstat -tlnp 와 동일 (ss가 최신)
ss -tlnp | grep 8080               # 특정 포트 확인
lsof -i :8080                       # 8080 포트 사용 중인 프로세스

# 연결 상태
ss -s                               # 소켓 통계 요약
ss -tn state ESTABLISHED           # 현재 수립된 TCP 연결
ss -tn | awk '{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn
# 외부 IP별 연결 수 확인 (DDoS 감지 등)

# DNS 조회
dig myapp.com                       # DNS 조회 (상세)
dig myapp.com +short                # IP만 출력
nslookup myapp.com                  # Windows 스타일
host myapp.com                      # 간단한 DNS 조회

# 연결 테스트
curl -v https://api.myapp.com/health        # HTTP 요청 + 상세 출력
curl -o /dev/null -s -w "%{http_code}\n" https://api.myapp.com/health  # 상태 코드만
wget -q --spider https://api.myapp.com      # 연결 확인

# 네트워크 경로 추적
traceroute google.com               # 경유 경로 추적
ping -c 4 google.com               # 연결 가능 여부 + 지연 시간

# TCP 연결 테스트 (방화벽 오픈 여부 확인)
telnet my-db.example.com 3306      # DB 포트 열려 있는지
nc -zv my-redis.example.com 6379  # Redis 포트 확인
```

## 4. 권한 시스템

### 4-1. 파일 권한

```bash
ls -l /app/start.sh
# -rwxr-x--- 1 ubuntu appgroup 1024 Jan 15 14:23 start.sh
#  ↑↑↑↑↑↑↑↑↑   ↑       ↑
#  권한 비트    소유자  소유 그룹

# 권한 비트: - rwx r-x ---
# 첫 문자: - (파일), d (디렉토리), l (심볼릭 링크)
# rwx: 소유자 권한 (읽기, 쓰기, 실행)
# r-x: 그룹 권한
# ---: 기타 권한

# 권한 변경
chmod 755 start.sh        # rwxr-xr-x (숫자)
chmod u+x start.sh        # 소유자에게 실행 권한 추가
chmod g-w config.yml      # 그룹의 쓰기 권한 제거
chmod -R 644 /app/config/ # 디렉토리 내 전체 재귀 적용

# 소유자 변경
chown ubuntu:appgroup start.sh       # 소유자:그룹 변경
chown -R appuser:appgroup /app/      # 재귀 변경

# 숫자 권한 계산
# r=4, w=2, x=1
# 7 = rwx (4+2+1)
# 6 = rw- (4+2)
# 5 = r-x (4+1)
# 4 = r-- (4)
# 644 = rw-r--r-- (소유자: 읽기쓰기, 그룹/기타: 읽기만)
# 755 = rwxr-xr-x (소유자: 전체, 그룹/기타: 읽기+실행)
```

### 4-2. 사용자 관리

```bash
# 현재 사용자 확인

# 사용자 생성

# /etc/sudoers 파일에 추가 (visudo 명령어로)

# 루트 권한으로 명령 실행
sudo systemctl restart nginx
sudo -i                             # root 쉘로 전환 (주의!)
```

## 5. systemd — 서비스 관리

```bash
# 서비스 상태 확인
systemctl status nginx
systemctl status mysql
systemctl status docker

# 출력 예시:
# ● nginx.service - A high performance web server
#    Loaded: loaded (/lib/systemd/system/nginx.service; enabled)
#    Active: active (running) since Mon 2024-01-15 09:00:00 KST
# → Active: active (running) = 정상 동작
# → Active: failed = 장애

# 서비스 시작/중지/재시작
systemctl start nginx
systemctl stop nginx
systemctl restart nginx
systemctl reload nginx   # 설정 리로드 (중단 없이)

# 부팅 시 자동 시작
systemctl enable nginx
systemctl disable nginx

# 로그 확인 (journald)
journalctl -u nginx                 # nginx 서비스 로그
journalctl -u nginx -n 50          # 최근 50줄
journalctl -u nginx --since "1 hour ago"
journalctl -u nginx -f             # 실시간 스트리밍
journalctl --since "2024-01-15 09:00" --until "2024-01-15 10:00"
```

## 7. 로그 분석 실전 패턴

```bash
# 특정 시간대 에러 로그
grep "2024-01-15 14:" /var/log/app/app.log | grep -i error

# 가장 많이 발생한 에러 메시지 Top 10
grep "ERROR" app.log | awk -F'ERROR' '{print $2}' | sort | uniq -c | sort -rn | head -10

# 최근 10분 동안 ERROR 급증 여부
grep "ERROR" app.log | awk '{print $1, $2}' | cut -c1-15 | uniq -c
# (분당 ERROR 건수 확인)

# 특정 사용자 관련 로그 (traceId 또는 userId로 추적)
grep '"userId":"12345"' app.log | tail -100

# 응답 시간이 2초 이상인 요청 찾기 (nginx access.log)
awk '$NF > 2' /var/log/nginx/access.log | head -20
# (마지막 컬럼이 응답 시간인 경우)

# 로그 파일 크기 확인 및 회전 상태
ls -lh /var/log/app/
du -sh /var/log/app/

# 실시간 로그 여러 파일 동시에
tail -f /var/log/app/*.log | grep -v DEBUG
```

## 8. SSH 접속과 보안

```bash
# SSH 접속
ssh ubuntu@52.78.xxx.xxx                    # 기본
ssh -i ~/.ssh/mykey.pem ec2-user@xxx.xxx.xxx.xxx  # 키 파일 지정
ssh -p 2222 ubuntu@xxx.xxx.xxx.xxx          # 포트 지정

# SSH 설정 파일 (~/.ssh/config)
Host myserver
  HostName 52.78.xxx.xxx
  User ubuntu
  IdentityFile ~/.ssh/mykey.pem
  Port 22

# 위 설정 후: ssh myserver

# 파일 전송
scp localfile.txt ubuntu@server:/remote/path/
scp -r localdir/ ubuntu@server:/remote/
rsync -avz --progress localdir/ ubuntu@server:/remote/dir/

# 포트 포워딩 (터널링)
# 로컬 5432 → 원격 서버를 통해 DB 3306으로
ssh -L 5432:private-db.internal:5432 ubuntu@bastion-server -N
# 이후 localhost:5432로 접속하면 private DB로 연결됨

# SSH 보안 설정 (/etc/ssh/sshd_config)
PermitRootLogin no              # root SSH 로그인 금지
PasswordAuthentication no       # 비밀번호 인증 금지 (키만 허용)
AllowUsers ubuntu appuser       # 특정 유저만 허용
Port 2222                       # 기본 포트 변경 (선택)
```

## 9. 자주 쓰는 유용한 명령어 모음

```bash
# 디스크 풀(Disk Full) 원인 찾기
df -h                              # 어느 파티션이 꽉 찼나
du -sh /var/log/* | sort -rh | head  # 큰 로그 파일 찾기
find / -size +1G -type f 2>/dev/null # 1GB 이상 파일 찾기

# 메모리 부족 원인 찾기
free -h
ps aux --sort=-%mem | head -10     # 메모리 많이 쓰는 프로세스
cat /proc/{PID}/status | grep VmRSS # 특정 프로세스 메모리 상세

# CPU 100% 원인 찾기
top -bn1 | head -20
ps aux --sort=-%cpu | head -10
# 특정 스레드 확인
ps -eLf | grep java | head -20

# 좀비 프로세스 확인
ps aux | awk '$8 == "Z"'

# 열린 파일 디스크립터 수
ls /proc/{PID}/fd | wc -l         # 특정 프로세스
lsof -p {PID} | wc -l
ulimit -n                          # 최대 허용 FD 수 (기본 1024)

# 현재 시간 & 타임존
```
